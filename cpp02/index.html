    cpp02 モジュール概要¶

    このモジュールは何？¶

    自分で作ったクラスを、int や float みたいに 自然に使えるようにする モジュールです。

    C 言語では a + b や a > b は int や float にしか使えませんでした。 C++ では 演算子オーバーロード という仕組みを使って、 自分で作ったクラスにも + や > を 使えるようにできます。

    C 言語:   fixed_add(a, b)   ← 関数呼び出しでしか書けない
C++:      a + b              ← 自然に書ける！

    このモジュールでは、「固定小数点数 (Fixed-point)」 という小さなクラスを少しずつ育てていきます。

  ↓
ex01: 数値を出し入れできるようにする
  ↓
ex02: + - * / などの演算子を追加する
  ↓
ex03: 実際に使って三角形の判定をする

    このモジュールで初めて出てくる道具¶

    OCF (Orthodox Canonical Form) って何？¶

    「C++ のクラスに必ず書く 4 つの関数」 のことです。 42 の課題では毎回このセットを要求されます。

    OCF の 4 つの関数
=================

1. デフォルトコンストラクタ  Fixed()
   → 空っぽのオブジェクトを作る

2. コピーコンストラクタ      Fixed(const Fixed &src)
   → 既にあるオブジェクトをコピーして新規作成

3. コピー代入演算子          operator=(const Fixed &rhs)
   → 既にあるオブジェクトに値を上書き

4. デストラクタ              ~Fixed()
   → オブジェクトが消える時に後片付け

    なぜ OCF が必要？¶

    C++ では、書かなくてもコンパイラが自動で作ってくれます。 でも、メンバ変数にポインタがあるクラスだと、 自動生成のコピーは「ポインタだけコピー」して 中身を共有してしまい、バグの元になります。

    自動生成のコピー (浅いコピー)
───────────────────────────
a の中: [ptr]─→[データ]
b = a したら:
a の中: [ptr]─→[データ]
b の中: [ptr]─┘  ← 同じ場所を指す！
→ a を delete すると b も壊れる

    だから、OCF を自分で書く習慣をつけておきます。

    演算子オーバーロードって何？¶

    + とか == とか << の動作を 自分で定義することです。

    // int には最初から + が使える
int x = 10 + 5;      // 15

// Fixed でも + を定義すれば使える
Fixed y = Fixed(10) + Fixed(5);  // 15

    関数の名前を operator+ のように書くと、 コンパイラが「+ が呼ばれた時にこれを使う」 と解釈してくれます。

    固定小数点 (Fixed-point) って何？¶

    小数を整数で表す方法です。

    float や double は計算のたびに微妙な誤差が出ますが、 固定小数点は整数で表現するので 同じ計算は必ず同じ結果になります。 ゲームや金融計算など、「誤差が許されない場面」 で使います。

      普通の整数:     10 はそのまま 10

  固定小数点:     10 を 256 倍して 2560 として保存
                  使う時は 256 で割って 10 に戻す

  ┌──────────────────────┬──────────┐
  │  整数部 (24 ビット)    │ 小数部    │
  │                      │ (8 ビット) │
  └──────────────────────┴──────────┘
         ↑                    ↑
     普通の数の部分      1/256 の精度

    小数部 8 ビット = 2^8 = 256 段階。 だから小数部を表す最小単位は 1/256 ≒ 0.00390625 です。

    共通ルール（全 exercise 共通）¶

    • コンパイル: c++ -Wall -Wextra -Werror -std=c++98
    • using namespace std; は 禁止
    • printf / malloc / free は 使用禁止
    • ヘッダファイルに関数の中身を書かない
    • friend キーワードは 禁止
    • クラス名は UpperCamelCase、メンバは _lowerCamel

    コピーコンストラクタ vs 代入演算子¶

    初心者が 一番混乱する ポイントです。 「新しく作る」か「上書きする」か の違いです。

    Fixed a;        // デフォルトコンストラクタ
                // （新しい空オブジェクト）

Fixed b(a);     // コピーコンストラクタ
                // （a をコピーして b を新規作成）

Fixed c;        // デフォルトコンストラクタ
c = a;          // 代入演算子
                // （既存の c に a の値を上書き）
    コピーコンストラクタ 代入演算子
    いつ呼ばれる？ 新しく作る とき 既にある ものに上書きするとき
    書き方の例 Fixed b(a); c = a;
    C で言うと memcpy で新規作成 memcpy で上書き

    C と C++ の比較表¶

    /* 初期化 */
struct Fixed s;
memset(&s, 0, sizeof(s));

/* コピー */
struct Fixed copy;
memcpy(&copy, &s, sizeof(s));

/* 足し算は関数呼び出し */
int r = fixed_add(a, b);

/* 後片付けも手動 */
free(ptr);

    // 初期化（自動で 0 になる）
Fixed a;

// コピー（自動でコピーされる）
Fixed b(a);

// 足し算が演算子で書ける
Fixed r = a + b;

// 後片付けも自動
// ← 何も書かなくていい！

    変わったこと一覧:

    C C++ 一言で言うと
    memset で初期化 コンストラクタ 自動化
    memcpy でコピー コピーコンストラクタ 自動化
    free を手動 デストラクタ 自動化
    fixed_add(a, b) a + b 自然な記法
