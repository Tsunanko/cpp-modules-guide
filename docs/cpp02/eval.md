## Exercise 別 Q&A

### Ex00: My First Class in Orthodox Canonical Form

| 質問 | 模範回答 | なぜ聞かれるか |
|------|---------|---------------|
| OCF とは何か？ | **4 つの関数を明示的に定義する規約**: (1) デフォルトコンストラクタ (2) コピーコンストラクタ (3) コピー代入演算子 (4) デストラクタ。コンパイラが自動生成するが、明示することで挙動を制御できる | **このモジュール最重要の概念** |
| なぜコピーコンストラクタが必要？ | コンパイラの自動生成は **shallow copy** (ビット単位コピー)。ポインタメンバがある場合、2 つのオブジェクトが同じメモリを指し、**ダブル free** になる。明示的に書くことで deep copy を実装できる | ポインタ管理の理解 |
| 自己代入チェックはなぜ必要？ | `a = a` の場合、チェックなしでポインタメンバを先に `delete` すると元データが消える。`if (this != &rhs)` で防ぐ | 安全なコーディング |
| コピーコンストラクタと代入演算子の違いは？ | **コピー ctor**: オブジェクトが**新規作成**される時 (`Fixed b(a)`)。**代入演算子**: **既存**オブジェクトに上書きする時 (`c = a`) | 呼び出しタイミングの区別 |
| `static const int` は何か？ | クラス全体で共有される定数。インスタンスごとに持たない。C の `static const` (ファイルスコープ) に似るが、クラスに紐づく | 設計意図の理解 |

!!! info "よく聞かれる口頭質問"
    **Q: デストラクタが逆順で呼ばれるのはなぜ？**

    A: C++ のスタック上のオブジェクトは **LIFO (後入れ先出し)** で破棄される。
    `a, b, c` の順に構築されたら `c, b, a` の順にデストラクタが呼ばれる。
    これは依存関係の安全性のため (後に作られたオブジェクトが先のオブジェクトに依存している場合がある)。

    **Q: 初期化子リストと本体での代入の違いは？**

    A: 初期化子リスト `: _value(0)` は**初期化** (オブジェクトの構築と同時)。
    本体 `{ _value = 0; }` は**代入** (一度デフォルト構築されてから上書き)。
    const メンバや参照メンバには初期化子リストが**必須**。効率も初期化子リストの方が良い。

### Ex01: Towards a more useful fixed-point number class

| 質問 | 模範回答 | なぜ聞かれるか |
|------|---------|---------------|
| 固定小数点数とは何か？ | 整数のビット列を「整数部 + 小数部」に分割して小数を表現する方式。8 ビットの小数部 = 1/256 の精度。浮動小数点と違い、精度が固定で**決定論的な演算**ができる | **このモジュールの核心概念** |
| `n << 8` は何をしている？ | 整数 n を 256 倍して固定小数点の内部表現に変換。小数部 8 ビット分の空間を確保する。例: `10 << 8 = 2560` → 内部値 2560 が実数 10.0 を表す | 変換の仕組み |
| なぜ `roundf` が必要？ | `float → int` の暗黙変換はデフォルトで**切り捨て**。`42.42 * 256 = 10859.52` を切り捨てると `10859` → `42.41796875` (誤差大)。`roundf` で四捨五入すると `10860` → `42.421875` (誤差小) | 精度の理解 |
| `operator<<` とは？ | `std::cout << fixed` を可能にする演算子オーバーロード。左オペランドが `std::ostream` なので**非メンバ関数**として定義。`friend` は使わず public の `toFloat()` 経由で実装 | 出力演算子の設計 |
| `42.42f` が `42.4219` になるのはなぜ？ | 8 ビットの小数部で表現できる最小単位は `1/256 = 0.00390625`。`42.42` に最も近い表現は `42.421875 (10860/256)`。これが固定小数点の精度限界 | 精度限界の理解 |

!!! info "よく聞かれる口頭質問"
    **Q: `static_cast<float>` と C スタイルキャスト `(float)` の違いは？**

    A: 機能は同じだが、`static_cast` は**意図が明確**で、コンパイラのチェックが厳しい。
    `const_cast`, `reinterpret_cast`, `dynamic_cast` と区別されるため、
    コードレビューで「何の変換か」が一目でわかる。

    **Q: なぜ `operator<<` に `friend` を使わないのか？**

    A: `toFloat()` が public なので、private メンバにアクセスする必要がない。
    `friend` は**本当に private アクセスが必要な場合**にのみ使うべき。
    そもそも 42 の課題では `friend` は禁止されている。

### Ex02: Now we're talking

| 質問 | 模範回答 | なぜ聞かれるか |
|------|---------|---------------|
| 前置と後置インクリメントの違いは？ | **前置 (`++x`)**: 変更後の自分を**参照**で返す。コピーなしで効率的。**後置 (`x++`)**: 変更前の値を**コピー**で返す。ダミー `int` 引数で区別 | **最頻出質問** |
| なぜ前置は参照で後置はコピーなのか？ | 前置は「変更後の自分」→ 参照で返せる。後置は「変更前の値」→ 変更前の値をローカル変数に保存し、本体を変更後にコピーを返す。ローカル変数の参照を返すとダングリング | 参照 vs 値の理解 |
| 算術演算で float 経由にする理由は？ | 乗算で `_value * _value` だとスケールが `256^2` になる。`2560 * 512 = 1310720` → `/ 256 = 5120` (正解は 20)。float 経由なら自動的に正しいスケール | 固定小数点の演算 |
| static min/max が 2 バージョンある理由は？ | `const` オブジェクトを渡した場合は const 版、非 `const` なら非 const 版が呼ばれる。戻り値の参照も const/非 const に合わせる必要がある | const 正確性 |
| 除算でゼロ除算した場合は？ | float 経由の除算なので `inf` または `nan` になる。subject で未定義なのでクラッシュしなければ OK | エッジケースの認識 |

!!! info "よく聞かれる口頭質問"
    **Q: 比較演算子で `toFloat()` を使わず `_value` を直接比較する理由は？**

    A: 両方のオブジェクトが同じスケール (256 倍) で格納されているため、
    内部値の大小関係がそのまま実数の大小関係と一致する。
    float 変換すると精度ロスの可能性があるため、直接比較の方が正確で効率的。

    **Q: ダミーの `int` 引数は何に使う？**

    A: コンパイラが前置と後置を区別するためだけの**仕様上の慣習**。
    実際の値は使わない。呼び出し時に `a++` と書くとコンパイラが自動的に `a.operator++(0)` を生成する。

### Ex03: BSP

| 質問 | 模範回答 | なぜ聞かれるか |
|------|---------|---------------|
| Point のメンバを const にする理由は？ | 座標は「一度決まったら動かない」性質を持つ。const にすることで**不変性を型レベルで保証**し、誤って書き換えるコードをコンパイルエラーにできる | const メンバの設計意図 |
| const メンバがあると代入演算子はどうなる？ | const メンバは代入できないため、`operator=` は**実質 no-op** になる。`(void)rhs;` で未使用引数の警告を抑制し、`return *this;` で自分を返す | OCF + const の相互作用 |
| BSP のアルゴリズムを説明して | 三角形の 3 辺それぞれに対して点との外積を計算。外積の符号が**全て同じ** (全て正 or 全て負) なら内部。**符号が混在**なら外部。外積が **0** なら辺上 (false) | アルゴリズムの理解 |
| 外積とは何か？ | 2D の外積 `(P1→P2) x (P1→P3)` は、P3 が P1→P2 の左にあれば正、右にあれば負、線上なら 0。これにより「辺のどちら側にいるか」を判定できる | 数学的背景 |
| コピーコンストラクタで `*this = src` が使えない理由は？ | `operator=` が no-op なので `_x`, `_y` がコピーされない。const メンバは初期化子リスト `: _x(src._x), _y(src._y)` でしか初期化できない | const メンバとコンストラクタ |

## 概念の深掘り

### 固定小数点数の仕組み (図解)

```
  Q24.8 フォーマット (32bit int, 小数部 8bit)
  ============================================

  ビット列:
  ┌──────────────────────────────┬──────────┐
  │ 31                        8 │ 7      0 │
  │      整数部 (24 bits)        │ 小数部    │
  │                             │ (8 bits) │
  └──────────────────────────────┴──────────┘

  精度: 1/256 = 0.00390625

  変換テーブル:
  ┌──────────┬──────────────┬──────────────────┐
  │  実数    │  内部値      │  計算式          │
  ├──────────┼──────────────┼──────────────────┤
  │  0.0     │  0           │  0 << 8          │
  │  1.0     │  256         │  1 << 8          │
  │  10.0    │  2560        │  10 << 8         │
  │  10.5    │  2688        │  roundf(10.5*256)│
  │  42.42   │  10860       │  roundf(42.42*256)│
  │  -1.0    │  -256        │  -1 << 8         │
  │  0.5     │  128         │  roundf(0.5*256) │
  │  0.00390625│  1          │  最小単位         │
  └──────────┴──────────────┴──────────────────┘

  float との比較:
  ┌────────────┬───────────────────┬────────────────────┐
  │            │  固定小数点 (Q24.8)│  float (IEEE 754)  │
  ├────────────┼───────────────────┼────────────────────┤
  │  精度      │  常に 1/256       │  値の大きさに依存   │
  │  範囲      │  -8388608〜8388607│  ±3.4 * 10^38     │
  │  決定論性  │  ✅ 常に同じ結果  │  ❌ 環境で異なる場合│
  │  演算速度  │  ✅ 整数演算      │  △ FPU 依存       │
  │  精度限界  │  0.00390625       │  約 7 桁の有効数字  │
  └────────────┴───────────────────┴────────────────────┘
```

### OCF の 4 要素 + 演算子オーバーロードの全体像

```
┌─────────────────── Fixed クラス ───────────────────┐
│                                                     │
│  OCF (正統派カノニカルフォーム):                      │
│    ① Fixed()              デフォルトコンストラクタ   │
│    ② Fixed(const Fixed&)  コピーコンストラクタ       │
│    ③ operator=            コピー代入演算子           │
│    ④ ~Fixed()             デストラクタ               │
│                                                     │
│  変換コンストラクタ (ex01):                          │
│    Fixed(int)  / Fixed(float)                       │
│    toInt()     / toFloat()                          │
│                                                     │
│  演算子オーバーロード (ex02):                        │
│    比較:    > < >= <= == !=                          │
│    算術:    + - * /                                  │
│    増減:    ++x x++ --x x--                          │
│    static:  min() max() (const/非const)              │
│                                                     │
│  非メンバ:                                          │
│    operator<<(ostream&, const Fixed&)                │
└─────────────────────────────────────────────────────┘
```

## 即不合格フラグ一覧

| フラグ | 条件 |
|--------|------|
| **Forbidden function** | `malloc`/`free`/`printf`/`using namespace`/`friend` |
| **Missing OCF** | 4 要素のいずれかが欠けている |
| **Invalid compilation** | `-Wall -Wextra -Werror -std=c++98` でコンパイルできない |
| **Wrong output** | subject の期待出力と不一致 (特に ex00 のメッセージ順序) |
| **Crash** | segfault やクラッシュ |
| **Can't support / explain code** | OCF の意味、固定小数点の仕組み、前置/後置の違いを説明できない |

!!! danger "「Can't support / explain code」に注意"
    コードが動いていても、**なぜそう書いたかを説明できなければ不合格**になりうる。
    特に以下は必ず答えられるようにしておくこと:

    - OCF の 4 要素とそれぞれの役割
    - 固定小数点の変換計算 (`<< 8`, `* 256`, `roundf`)
    - 前置インクリメントが参照で後置がコピーである理由
    - const メンバがある場合の代入演算子の挙動
