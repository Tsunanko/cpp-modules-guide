    用語集¶

    C の知識をベースに、C++ 用語を最短で理解するための辞書。

    メモリ基本用語¶

    ゴミ値 (garbage value)
    初期化していない変数に入っている、予測できない値のこと。
    C / C++ で変数を宣言しただけで使うと、メモリに残っていた前のデータの残骸が
    そのまま読み取られる。毎回違う値が出るためバグの温床になる。
    ```c int x; // 初期化なし → ゴミ値 printf("%d\n", x); // 謎の数字が出る (毎回違う)
    int y = 0; // 明示的に初期化 → ゴミ値を防げる
    ```
    対策: 宣言と同時に初期化する、または C++ のコンストラクタを使う。
    ヒープ (heap)
    プログラム実行中に動的に確保するメモリ領域。new / malloc で確保、delete / free で解放する。
    スタック (stack)
    関数呼び出しに合わせて自動管理されるメモリ領域。ローカル変数が置かれる。関数終了で自動解放。
    メモリリーク (memory leak)
    new / malloc で確保したメモリを解放し忘れて、使われないメモリが残り続けること。
    42 の評価ではフラグ扱いで即 0 点。
    ダングリングポインタ (dangling pointer)
    すでに解放された（または消滅した）メモリを指したままのポインタ。アクセスするとクラッシュや未定義動作。

    クラス / オブジェクト¶

    クラス (class)
    データ (メンバ変数) と処理 (メンバ関数) をまとめた「型」。C の struct に関数をぶら下げて可視性を足したもの。
    cpp class Foo { ... };
    オブジェクト / インスタンス
    クラスから作られた実体。
    cpp Foo f; // f は Foo クラスのオブジェクト
    メンバ変数
    クラスが持つデータ。42 慣習で _ プレフィックス。
    メンバ関数
    クラスに属する関数。obj.func() で呼ぶ。

    特殊メンバ関数¶

    コンストラクタ (constructor)
    オブジェクト生成時に自動で呼ばれる特殊関数。クラス名と同じ名前。
    cpp Foo(); // デフォルトコンストラクタ Foo(int x); // 引数ありコンストラクタ
    デストラクタ (destructor)
    オブジェクト破棄時に自動で呼ばれる。~ + クラス名。
    cpp ~Foo();
    デフォルトコンストラクタ
    引数なしのコンストラクタ。Foo f; で呼ばれる。
    private に置くと生成を禁止できる (ex02 で使用)。
    メンバ初期化子リスト
    コンストラクタの : 以降。本体 {} より先にメンバを初期化する。
    cpp Foo::Foo() : _x(0), _y(0) {}

    アクセス指定子¶

    public
    クラス外から自由にアクセス可能。
    private
    クラス内からのみアクセス可能。デフォルト (class)。
    protected
    派生クラスからアクセス可能。cpp03 以降で使用。

    const 三態¶

    const 修飾 (型)
    値を書き換えない。
    cpp const int n = 5;
    const 参照 (引数)
    コピーせず読み取り専用で受け取る。定石。
    cpp void foo(const std::string &s);
    const メンバ関数 (関数末尾)
    メンバ変数を変更しないことを保証。const Foo から呼び出せる唯一の関数。
    cpp int getX() const { return _x; }

    参照とポインタ¶

    参照 (reference, &)
    別名。NULL にならず、宣言時に対象必須。* や -> 不要。
    cpp int x = 5; int &r = x; // r は x の別名 r = 10; // x == 10
    ポインタ (pointer, *)
    C と同じ。NULL 可、再代入可、* / -> が必要。

    名前空間¶

    名前空間 (namespace)
    名前衝突を防ぐ「箱」。std:: で標準ライブラリにアクセス。
    スコープ解決演算子 (::)
    「どのスコープに属するか」を示す。
    • std::cout — std 名前空間の cout
    • Foo::bar() — Foo クラスの bar
    • ::global_var — グローバルスコープ

    static¶

    static メンバ変数
    全インスタンスで共有される変数。.cpp で実体定義が必要。
    cpp // hpp static int _count; // cpp int Foo::_count = 0;
    static メンバ関数
    インスタンスなしで呼べる関数。this を持たない。
    cpp Foo::getCount();
    static (関数内)
    C と同じ。関数呼び出しをまたいで値を保持。
    static (ファイル内)
    C と同じ。そのファイル内からのみ使用可能。main.cpp の static void doAdd() など。

    I/O ストリーム¶

    std::cout
    標準出力ストリーム。printf の代替。
    std::cerr
    標準エラー出力 (バッファなし)。
    std::cin
    標準入力。scanf の代替。
    << 出力演算子
    ストリームに値を送る。型ごとにオーバーロード済み。
    >> 入力演算子
    ストリームから値を取り出す。空白で区切る。
    std::endl
    改行 + バッファフラッシュ。"\n" は改行のみ。
    std::getline(std::cin, s)
    改行までを s に読み込む (改行は破棄)。
    <iomanip>
    書式指定ヘッダ。std::setw(), std::right, std::setfill() など。

    std::string¶

    std::string
    自動メモリ管理の文字列クラス。char* の悩みから解放される。
    主なメンバ関数
    • .length() / .size() — 長さ
    • .empty() — 空判定
    • .substr(pos, len) — 部分文字列
    • .find(str) — 検索
    • .c_str() — C の const char* に変換
    演算子
    • + / += — 連結
    • == / != — 比較
    • [] — 文字アクセス

    Makefile / コンパイル¶

    -Wall -Wextra -Werror
    警告フル、かつ警告を即エラーに。
    -std=c++98
    42 指定の古い C++ 規格。auto, ラムダ式等は使えない。
    c++
    macOS/Linux 標準の C++ コンパイラ (clang or g++ のシンボリックリンク)。
